Issue Ticketing System

Web Application

Scope and limitation
Currently HMPC using an excel to monitor and make a daily, weekly and monthly report that will be represented to the VP of Comptroller every meeting. The main objective of the system is to handle requests for users from various department which will assist by staff. We proposed a web-based trouble ticketing system that can be used as a bridge between the users and the technical support team at ITSD (Information Technology Services Department) of HMPC (Hino Motors Philippines Corporation). In order to provide consistent and quality support to the users. In the proposed system we are going to track the information and problems that come from the users into the Help Desk. In addition, assign the problems to the appropriate staff make sure that the problems are solved. The status of Issue can be monitor by New, Pending or Closed. Can specify a priority to his/her ticket like urgent or normal. Can issue a problem on mobile. The system allows them to track problems by assigned person, for a specific user and many other ways. Has a login page for technical support staff to stop intruders. The system can assist the users in HMPC to find the solutions for routine questions through FAQ part. Generation of issue reports daily, weekly, and monthly. The system does not include the user who does not log their problem.

Problem encountered
Takes time to generate report using excel. There was no standard and organized process for handling the users' requests. Sometimes no one responded right away and other times multiple people did, showing that we were not very coordinated.

Benefits of System
Improved technical support team quality as all inquiries or problems are immediately taken action timely identification, diagnosis and resolution of problem Improved user or customer satisfaction.

Technologies used
ASP.NET MVC 5, SignalR, Microsoft SQL Server, Bootstrap
